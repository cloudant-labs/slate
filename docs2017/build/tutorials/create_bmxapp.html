Creating a simple Bluemix application to access a Cloudant database

This tutorial shows you how to create an IBM® Bluemix® application that uses the Python programming language to access an IBM® Cloudant® NoSQL DB for Bluemix® database, hosted in your Bluemix service instance.

Context

A significant advantage of Bluemix is that you can create and deploy applications within Bluemix itself. You do not have to find and maintain a server to run your applications.

If you are already using a Cloudant NoSQL DB database instance within Bluemix, it makes sense to have your applications there, too.

Bluemix applications are typically created by using Cloud Foundry technology. Cloud Foundry offers a Platform-as-a-Service (PaaS) capability that simplifies the process of creating applications that can be deployed and run within a Cloud environment.

A separate tutorial showed you how to create a stand-alone Python application that uses a Cloudant NoSQL DB database instance within Bluemix. In this tutorial, you set up and create a small Python application hosted within Bluemix. The application connects to your Cloudant NoSQL DB database instance, and creates a single, simple document.

Python code specific to each task is provided as part of this tutorial. A complete Python program, sufficient to demonstrate the concepts, is provided in the tutorial, here.

No attempt was made to create efficient Python code for this tutorial; the intention is to show simple and easy-to-understand working code that you can learn from and apply for your own applications.

Also, no attempt was made to address all possible checks or error conditions. Some example checks are included to illustrate some of the techniques, but you should apply normal best practices for checking and handling all warnings or error conditions that are encountered by your own applications.

Task overview

To create a working Python application on Bluemix that can access a Cloudant NoSQL DB database instance, you need to perform the following tasks:

  • Create a Python application environment on Bluemix.
  • Ensure that the Python application environment has a 'connection' to a Cloudant NoSQL DB database instance.
  • (One-off task) Download and install the Cloud Foundry and Bluemix command line toolkits.
  • Download the 'starter' application.
  • Customize the starter application to create your own application for accessing the Cloudant NoSQL DB database instance.
  • Upload your application and test that it works.
  • Perform basic application maintenance tasks.
  • Diagnose and resolve problems (troubleshooting).

  4. Uploading and running your Application
  5. Maintaining and troubleshooting your Application
