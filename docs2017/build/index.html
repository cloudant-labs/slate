Getting started with Cloudant

IBM® Cloudant® NoSQL DB for Bluemix® is a document-oriented DataBase as a Service (DBaaS). It stores data as documents in JSON format. It's built with scalability, high availability, and durability in mind. It comes with a wide variety of indexing options including map-reduce, Cloudant Query, full-text indexing, and geospatial indexing. The replication capabilities make it easy to keep data in sync between database clusters, desktop PCs, and mobile devices.

For more information on all the Cloudant NoSQL DB offerings, see the main Cloudant NoSQL DB site. For more details on Cloudant NoSQL DB concepts, tasks and techniques, see the Cloudant NoSQL DB documentation.

You can launch the Cloudant NoSQL DB console from the service launch page on the Bluemix dashboard.

To use the service, follow these steps:

  1. Create a service instance using the Bluemix dashboard.
  2. Review the service instance access credentials, similar to the following example:
    {
    "cloudantNoSQLDB": {
        "name": "Cloudant-3s",
        "label": "cloudantNoSQLDB",
        "plan": "shared",
        "credentials": {
            "username": "someusername",
            "password": "secret",
            "host": "myhost-bluemix.cloudant.com",
            "port": 443,
            "url": "https://someusername:secret@myhost-bluemix.cloudant.com"
        }
    }
}

  3. Create applications that connect to the service instance, and which use the service credentials. These applications can be created by using various environments, including:

    • Mobile platforms
    • Java
    • Node.js
    • Python
    • Swift

    For more information, see the Cloudant homepage and the list of Client Libraries.

Tutorials are available that describe these tasks in more detail. More information about tasks such as authenticating with database instances, and querying data, is available in the API reference.
